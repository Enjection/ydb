## Быстрый старт (Перенаправленный контент)

Для полного руководства по быстрому старту см. [новую централизованную документацию](../../../../concepts/backup-collections.md).

```sql
-- 1. Создать коллекцию
CREATE BACKUP COLLECTION `my_backups`
    ( TABLE `/Root/db/table1`, TABLE `/Root/db/table2` )
WITH ( STORAGE = 'cluster', INCREMENTAL_BACKUP_ENABLED = 'true' );

-- 2. Создать полную резервную копию
BACKUP `my_backups`;

-- 3. Создать инкрементальные резервные копии
BACKUP `my_backups` INCREMENTAL;
```

## Новая структура документации

Документация по коллекциям резервных копий была реорганизована для устранения дублирования и улучшения удобства использования:

- **Единый источник истины**: Каждая часть информации существует в одном месте
- **Лучшая организация**: Четкое разделение между концепциями, справочниками и практическими руководствами  
- **Полное покрытие**: Все аспекты коллекций резервных копий задокументированы
- **Легкое обслуживание**: Обновления нужно делать только в одном месте
